Company Password Policy – Version 1.0
Document ID: password_policy
Department: Information Security
Effective Date: January 1, 2024
Version: v1
Classification: Internal – Confidential
1. Purpose
The purpose of this policy is to establish minimum requirements for the creation, use, and management of passwords to protect company systems, applications, and data from unauthorized access.
2. Scope
This policy applies to:
All employees
Contractors
Temporary staff
Third-party users with access to company systems
It covers all systems, including internal applications, cloud services, VPNs, and databases.
3. Password Requirements
All passwords must comply with the following requirements:
Minimum length: 12 characters
Must include at least:
One uppercase letter (A–Z)
One lowercase letter (a–z)
One number (0–9)
One special character (! @ # $ % ^ & *)
Must not contain:
Usernames
Employee ID numbers
Dictionary words
Previously used passwords
4. Password Expiration and Rotation
Passwords must be changed every 90 days
Users will receive a notification 7 days before expiration
Reuse of the last 5 passwords is prohibited
5. Multi-Factor Authentication (MFA)
MFA is mandatory for:
VPN access
Admin accounts
Cloud management consoles
Approved MFA methods include:
Authenticator applications
Hardware security keys
6. Password Storage and Handling
Passwords must never be:
Written down
Shared via email or messaging apps
Passwords must be stored only in:
Approved password managers
Plain-text password storage is strictly prohibited
7. Compromised Passwords
If a password is suspected to be compromised:
Change the password immediately
Notify the IT Security team within 1 hour
Review recent account activity
8. Administrative Accounts
Admin passwords must:
Be at least 16 characters
Be unique per system
Shared admin accounts are prohibited unless explicitly approved
9. Policy Violations
Failure to comply with this policy may result in:
Access revocation
Disciplinary action
Termination of employment
10. Review and Updates
This policy is reviewed annually
Updates require approval from the Information Security team
New versions supersede previous versions
Approved by: Head of Information Security
